# /editorconfig

Analisa o projeto e gera um `.editorconfig` com boas práticas, cobindo todos os tipos de arquivo relevantes.

## Usage

```
/editorconfig
/editorconfig --tabs        # usa tabs ao invés de spaces
/editorconfig --indent 4    # tamanho de indentação (default: 2)
```

## Phase 1: Detect file types (1 bash command)

```bash
find . -maxdepth 4 -type f -not -path "*/node_modules/*" -not -path "*/.git/*" -not -path "*/dist/*" -not -path "*/.next/*" | sed 's/.*\.//' | sort -u | head -40
```

With the extension list, identify which sections to include:
- `ts`, `tsx`, `js`, `jsx`, `mjs`, `cjs` → JavaScript/TypeScript
- `py` → Python
- `rs` → Rust
- `go` → Go
- `java`, `kt` → JVM
- `cs` → C#
- `rb` → Ruby
- `php` → PHP
- `css`, `scss`, `sass`, `less` → Stylesheets
- `html`, `htm`, `svelte`, `vue` → Templates
- `json`, `jsonc` → JSON
- `yaml`, `yml` → YAML
- `toml` → TOML
- `md`, `mdx` → Markdown
- `sh`, `bash`, `zsh` → Shell scripts
- `sql` → SQL
- `xml` → XML
- `Makefile`, `makefile` → Makefiles

Check if a `.editorconfig` already exists. If so, read it and update rather than overwrite.

## Phase 2: Write `.editorconfig`

Generate based on detected file types. Only include sections for types actually present in the project.

```editorconfig
# .editorconfig — generated by devrails /editorconfig
# https://editorconfig.org

root = true

# ── Universal defaults ──────────────────────────────────────────────────────
[*]
indent_style = space
indent_size = 2
end_of_line = lf
charset = utf-8
trim_trailing_whitespace = true
insert_final_newline = true

# ── Markdown — trailing spaces are significant (hard line breaks) ────────────
[*.{md,mdx}]
trim_trailing_whitespace = false
max_line_length = off

# ── YAML — always 2 spaces, spec requires it ────────────────────────────────
[*.{yaml,yml}]
indent_size = 2

# ── JSON ─────────────────────────────────────────────────────────────────────
[*.{json,jsonc}]
indent_size = 2

# ── Makefiles — tabs are required by the syntax ─────────────────────────────
[{Makefile,makefile,*.mk}]
indent_style = tab

# ── Shell scripts ────────────────────────────────────────────────────────────
[*.{sh,bash,zsh}]
indent_size = 2

# ── Python — PEP 8 ───────────────────────────────────────────────────────────
[*.py]
indent_size = 4
max_line_length = 88

# ── Go — gofmt uses tabs ─────────────────────────────────────────────────────
[*.go]
indent_style = tab
indent_size = 4

# ── Rust ─────────────────────────────────────────────────────────────────────
[*.rs]
indent_size = 4
max_line_length = 100

# ── Java / Kotlin ─────────────────────────────────────────────────────────────
[*.{java,kt,kts}]
indent_size = 4

# ── C# ────────────────────────────────────────────────────────────────────────
[*.{cs,csx}]
indent_size = 4

# ── SQL ───────────────────────────────────────────────────────────────────────
[*.sql]
indent_size = 2

# ── XML / SVG ────────────────────────────────────────────────────────────────
[*.{xml,svg,xsd,xsl}]
indent_size = 2
```

Adapt: apply `--tabs` flag by changing `indent_style = space` to `indent_style = tab` in `[*]`. Apply `--indent N` by replacing the default `indent_size`. Override per-language sections only when their canonical style conflicts with the flag.

## Phase 3: Explain key rules

After writing the file, print a brief explanation of non-obvious rules only (skip self-explanatory ones):

- **`root = true`** — stops EditorConfig from searching parent directories; always required at repo root
- **`end_of_line = lf`** — normalizes line endings cross-platform; prevents noisy diffs on Windows
- **`trim_trailing_whitespace = false` for Markdown** — trailing two spaces create a hard line break in Markdown; trimming them breaks formatting
- **`indent_style = tab` for Makefiles** — tab is syntactically required; spaces cause `make` to fail
- **`indent_style = tab` for Go** — `gofmt` enforces tabs; overriding to spaces will conflict with the formatter
- **`max_line_length = 88` for Python** — matches Black formatter default

## Rules

- Only include sections for file types actually detected in the project.
- Never invent sections for types not present.
- If a `.editorconfig` already exists, preserve any manual customizations and only add/update sections for new file types.
- Language-canonical styles (Go tabs, Makefile tabs) always override the global `--tabs`/`--indent` flags.
